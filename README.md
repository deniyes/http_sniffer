openvpn_sniffer
============================
sniffer http request and response from interfaces

```mermard
graph LR
    subgraph 拓客 (Lead Generation)
        A[确定目标客户画像] --> B{渠道选择困难？};
        B -- 渠道太多，不知如何选择 --> C[多渠道小范围测试];
        B -- 目标客户触达困难 --> D[内容营销/社交媒体互动/行业活动];
        C --> E{效果评估与优化};
        E -- 效果不佳 --> B;
        E -- 效果良好 --> F[初步筛选与意向确认];
        F -- 客户意愿不明确 --> G[提供价值内容/案例分析];
    end

    subgraph 测试 (Testing/Proof of Concept)
        F --> H[需求沟通与方案初步定制];
        H --> I{测试方案设计困难？};
        I -- 方案不贴合客户需求 --> J[深入了解客户业务痛点];
        I -- 测试成本过高 --> K[小范围/MVP测试];
        J --> L[执行测试/POC];
        K --> L;
        L --> M{测试结果评估困难？};
        M -- 数据收集不全面 --> N[完善数据埋点与监控];
        M -- 结果解读偏差 --> O[与客户共同制定评估标准];
        N --> P[测试报告与结果展示];
        O --> P;
        P -- 客户对结果有疑虑 --> Q[提供详细数据/成功案例对比];
    end

    subgraph 谈判 (Negotiation)
        P --> R[初步报价与方案沟通];
        R --> S{价格异议过大？};
        S -- 客户预算有限 --> T[调整方案/突出价值/分期付款];
        S -- 竞争对手价格更低 --> U[强调差异化优势/提供增值服务];
        T --> V{条款细节谈判困难？};
        U --> V;
        V -- 合同条款模糊不清 --> W[明确合同细节/法律顾问介入];
        V -- 双方立场差距大 --> X[寻找共同利益点/灵活调整条款];
        W --> Y[合同草拟与修改];
        X --> Y;
        Y --> Z{内部审批流程复杂？};
        Z -- 审批环节过多/时间过长 --> AA[优化内部流程/明确审批权限];
        AA --> BB[最终合同确认];
    end

    subgraph 下单 (Order Placement)
        BB --> CC[客户签字确认];
        CC --> DD{回款周期过长？};
        DD -- 客户付款意愿不高 --> EE[提供多种支付方式/优惠政策];
        DD -- 客户流程复杂 --> FF[积极配合客户流程/提供便捷支付渠道];
        EE --> GG[订单生效与执行];
        FF --> GG;
        GG --> HH[客户成功与持续服务];
        HH -- 客户流失风险 --> II[定期回访/提供增值服务/建立长期合作关系];
    end
```
